# 云平台架构设计

## 背景

挖财技术能力&系统产品化体系化，上云开放，对外输出。要求做到多系统产品形态聚合收敛，标准能力去重统一，系统和功能支持插拔。

## 目标

+ 跨系统公共能力（配置、路由、鉴权、账户等）收敛沉淀为公共服务
+ 子系统可动态注册
+ 子系统可独立开发部署
+ 数据源统一（数据网关）、元数据（权限、角色，菜单等）统一
+ 业务单元与系统解耦与数据耦合，能系统使用
+ 已有系统支持平滑迁移至新架构
